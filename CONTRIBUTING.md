# Contributing

Thanks for your interest in Fractal Intelligence Lab.

## Ways to contribute
- Add new fractal kernels.
- Improve the Fbit architecture.
- Add benchmarks and baselines.
- Improve documentation and examples.
- Fix tests, edge cases, and numerical stability issues.
- Build visualization tools.

## Workflow
1. Open an issue describing the proposal or bug.
2. Keep changes modular and small.
3. Add tests for new behavior.
4. Document assumptions clearly.
5. Submit a pull request.

## Research standards
- Separate established algorithms from experimental hypotheses.
- Avoid vague claims.
- Prefer reproducible experiments.
- Document parameter choices and random seeds.

## Code style
- Keep functions small and explicit.
- Use type hints where practical.
- Favor readability over cleverness.
- Document any numerical instability.

## Experimental additions
When adding a new module, please state:
- what it does,
- what it assumes,
- what baseline it should be compared against,
- what benchmark would evaluate it.

## Code of conduct
Please follow `CODE_OF_CONDUCT.md`.
