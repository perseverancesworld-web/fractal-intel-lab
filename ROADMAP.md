# Roadmap

## Phase 0: Internal prep
- Finalize the core skeleton: `core/`, `examples/`, `benchmarks/`, `tests/`.
- Write initial docs: `README.md`, `docs/architecture.md`, `docs/math-foundations.md`.
- Produce 2–3 minimal demos:
  - language embedding demo,
  - time-series demo,
  - visualization demo.

## Phase 1: Public GitHub launch
- Publish the repository.
- Add issue templates and contribution guidelines.
- Add CI for tests and linting.
- Share a short demo video.

## Phase 2: Benchmark release
- Publish robustness, memory retention, replay, and clustering benchmarks.
- Include plots and tables in `benchmarks/` and `docs/`.
- Document failure cases and negative results.

## Phase 3: Research draft
- Write a full white paper.
- Release arXiv preprint.
- Solicit outside review from researchers in dynamical systems, ML, and unconventional computation.

## Phase 4: Community growth
- Invite external contributors.
- Add new fractal kernels.
- Add new benchmark tasks.
- Improve visualization tooling.

## Phase 5: Extended research
- Compare against baseline systems.
- Study stability, interpretability, and memory dynamics.
- Explore sensor, language, and synthetic time-series inputs.
